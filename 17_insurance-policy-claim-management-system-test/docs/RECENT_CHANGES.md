# Recent Backend Changes (Senior Review Fixes)

This document outlines the recent improvements and bug fixes applied to the backend system based on the senior architectural review.

## 1. Financial Precision & Data Integrity
- **`Double` to `BigDecimal` Conversion:** Replaced all monetary fields (`claimAmount`, `premiumAmount`, `coverageAmount`, `totalPremiumPaid`, `amount`) across Entities, DTOs, and Repositories from `Double` to `BigDecimal` (with precision=15, scale=2). This prevents floating-point rounding errors which are unacceptable in financial/insurance applications.

## 2. Business Logic & Guardrails
- **Policy Cancellation Guard:** Added a strict check in `PolicyServiceImpl.cancelPolicy()` that prevents cancelling any policy if it has active claims (`SUBMITTED`, `UNDER_REVIEW`, `RECOMMENDED_FOR_APPROVAL`). This prevents "orphaned" claims against cancelled policies.
- **Start Date Validation:** Added validation in `purchasePolicy()` and `issuePolicy()` to reject any policy start dates that are in the past.
- **Overpayment Prevention Fix:** Fixed the `PremiumPaymentServiceImpl` overpayment guard. It now correctly compares the total payments against the total required premium (`premiumAmount * duration`) rather than just the `coverageAmount`.

## 3. Security Hardening
- **OTP Hashing:** OTPs (`emailOtp`, `phoneOtp`) are now securely hashed using `BCryptPasswordEncoder` before being stored in the database by the `OtpService`. They are verified using `passwordEncoder.matches()`. This prevents exposure of valid OTPs in case of a database breach.
- **Claim Document Validation:** Added strict file type validation (allowing only `image/jpeg`, `image/png`, `application/pdf`) and file size limits (max 10MB per file) in `ClaimDocumentServiceImpl` to prevent malicious or excessively large uploads.

## 4. Performance & Database Optimization
- **EAGER Fetching Fix:** Added `FetchType.LAZY` to the `@ManyToOne` relationship between `PolicyPlan` and `InsuranceProduct` to prevent N+1 query performance issues during pagination.
- **Repository Cleanup:** Deleted the accidentally duplicated and misspelled `InsurenceProductRepository`.

## 5. Code Quality & Maintainability
- **Shared Pagination Utility:** Created a centralized `PaginationValidator` utility class to standardize page size and sorting validation, reducing duplicated logic across service classes.
- **Logging Standardization:** Replaced manual `LoggerFactory` declarations with Lombok's `@Slf4j` in `CustomerServiceImpl` to maintain consistency across the codebase.
- **Boolean Wrapper Fixes:** Fixed compilation errors in `AuthServiceImpl` stemming from type mismatches between primitive `boolean` and wrapper `Boolean` fields generated by Lombok.
